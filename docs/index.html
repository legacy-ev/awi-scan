awi-scan
GitHub Validation v0.1.0
AI GitHub Actions security

Scan AI workflows for prompt-to-agent injection.

awi-scan is an open-source CLI and GitHub Action that finds where untrusted GitHub issue, PR, and comment text can steer an AI agent, then reach shell commands or repository mutations.

Vulnerable pattern

- id: triage
  uses: anthropics/claude-code-action@v1
  with:
    prompt: |
      Triage this issue:
      ${{ github.event.issue.body }}

- run: gh issue edit "$NUMBER" \
    --add-label "${{ steps.triage.outputs.labels }}"

awi-scan output

[HIGH] AWI001 P2A
Untrusted GitHub event text reaches an AI-agent prompt

Source: issue body
Sink: anthropics/claude-code-action@v1

[CRITICAL] AWI003 P2S
AI-agent output reaches a script or GitHub mutation step

Fix: validate agent output against a strict schema
and require approval before GitHub mutations.

What it detects

AWI001 P2A

Untrusted GitHub text reaches an AI-agent prompt.

AWI002 P2S

Untrusted event text reaches shell, gh, github-script, or API calls.

AWI003 P2S

AI-agent output reaches script execution or repository mutation.

AWI004 CONFIG

External-user triggers combine with write-token AI workflows.

Run it locally

npx awi-scan
npx awi-scan . --format markdown
npx awi-scan . --format sarif --output awi.sarif

Use it in CI

- uses: actions/checkout@v4
- uses: legacy-ev/awi-scan@v0.1.0
  with:
    format: markdown
    fail-on: high
